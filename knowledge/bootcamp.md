## Currículum por semana

### Semana 1 — Agent architecture + SDLC (cubre D1 de GH-600)
- Por qué Claude Code no basta: separar agente, harness y runtime
- CLAUDE.md como constitución del proyecto
- Pre-flight checks antes de cada interacción
- Lab: armar el primer CLAUDE.md para un repo regulado

### Semana 2 — Tools + environment (cubre D2 de GH-600)
- Skills, Hooks y MCP servers como capas del harness
- Sandboxing y aislamiento por entorno
- Lab: skill propio + hook de validación

### Semana 3 — Memory, state, execution (cubre D3 de GH-600)
- Manejo de memoria efímera vs persistente
- Estado distribuido en pipelines
- Lab: agente con estado en Postgres + replay determinístico

### Semana 4 — Eval + error analysis (cubre D4 de GH-600)
- Métricas para agentes: DORA + harness-specific
- Análisis de logs y trazas con Langfuse
- Lab: dashboard de métricas del propio repo del estudiante

### Semana 5 — Multi-agent orchestration (cubre D5 de GH-600)
- Workflows multi-agente en GitHub Actions
- Patrones: supervisor, swarm, hierarchical
- Lab: pipeline multi-agente para code review automatizado

### Semana 6 — Guardrails + responsibility (cubre D6 de GH-600)
- OWASP Top 10 for Agentic Applications 2026
- Auditoría y trazabilidad para reguladores
- Casos: banca, seguros, casino
- Lab: harness con guardrails completos + reporte de auditoría
